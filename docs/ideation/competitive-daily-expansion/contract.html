Context · 01

The problem

01merge_loop is a polished but solitary single-player daily puzzle: every player gets one deterministic board per day (seeded from a SHA256 of the date) and plays it alone. There is no reason to return beyond personal best, no social pull, and no way to compare your run against anyone else's. Engagement, reach, and session count per day are all left on the table.

02Separately, the AdMob configuration is nearly correct but has a release-blocking defect: the real ad unit IDs are wired up and useTestAds is false, yet the iOS GADApplicationIdentifier in Info.plist still holds Google's TEST App ID. On iOS this mismatch (real unit IDs + test App ID) prevents ads from serving and can get the app flagged.

03The goal is to convert the solo daily puzzle into a competitive, social, habit-forming daily game - difficulty tiers, global and friends leaderboards, identity, and retention mechanics - while keeping infrastructure cost at $0 (Supabase free tier) and shipping a cheat-resistant leaderboard from day one.

Commit · 02

Goals

1 Fix the iOS AdMob App ID so ads serve correctly on both platforms.
2 Add 4 difficulty tiers (easy=10, medium=8, hard=6, legendary=4 starting tiles; 30 moves all tiers), each a separate deterministic daily board playable once per tier per day.
3 Seed boards and leaderboards by UTC date so 'everyone plays the same board today' holds globally.
4 Ship a global per-tier daily leaderboard with server-side replay-verified scores (cheating effectively impossible).
5 Give every player an identity via Supabase anonymous auth + display name with zero signup friction.
6 Add a friends/local leaderboard built from friend codes + contacts matching, plus outbound score sharing to any platform.
7 Add retention mechanics: streaks + local daily-reminder notifications, achievements/badges, extended (weekly/monthly/all-time) leaderboards, and a practice mode.
8 Keep backend cost at $0 on Supabase free tier; design for a clean upgrade path as player count grows.
Signal · 03

Done when…

12 signals
  • 01 On a real iOS build, real ads serve and Info.plist GADApplicationIdentifier is the real App ID (not ...3940256099942544).
  • 02 Selecting each of the 4 tiers loads a distinct, deterministic board; the same tier on the same UTC date yields an identical board for every player; tile counts are 10/8/6/4.
  • 03 A player can complete each tier exactly once per UTC day; a second attempt that day is blocked.
  • 04 Submitting a completed run posts the move sequence; the Supabase edge function replays it against the regenerated (date,tier) board and writes a score that matches the client's locally computed score for legitimate runs.
  • 05 A fabricated or illegal move sequence is rejected by the edge function and never reaches the leaderboard.
  • 06 The global daily leaderboard for a given tier shows ranked display names + scores and resets at 00:00 UTC.
  • 07 A player can generate a friend code/link, a second device can redeem it, and both appear on each other's friends leaderboard.
  • 08 Contacts matching (with permission granted) surfaces at least known opted-in test contacts; denying permission degrades gracefully.
  • 09 A local notification fires at the configured daily time and when a streak is about to expire.
  • 10 Daily streak increments on consecutive-day completion and resets after a missed UTC day; achievements unlock on their trigger conditions.
  • 11 Cross-language test: the TypeScript PRNG and seeder produce byte-identical output to the Dart implementations for a battery of seeds.
  • 12 flutter analyze is clean and flutter test passes, including new engine/seeder/tier tests.

MVP — must ship ×9
  • Fix iOS AdMob App ID in Info.plist— Release-blocking defect; one-line fix, no reason to defer.
  • UTC-date seeding (replace local-date) with local countdown to reset— Prerequisite for a fair global 'same board for all' leaderboard.
  • 4 difficulty tiers via per-tier starting fill (10/8/6/4), 30 moves all tiers— Core game expansion; tile-count-only keeps it deterministic and balanceable.
  • Per-(date,tier) seeding, snapshots, and stats; once-per-tier-per-day rule— Each tier is its own daily puzzle and its own competition.
  • Record the move sequence per run— Required input for server-side replay verification; cheap to add now.
  • Supabase project + schema + RLS (players, scores)— Foundation for all online features at $0.
  • Anonymous auth + display name (with optional avatar/emoji)— Zero-friction identity maximizes leaderboard participation and ad views.
  • Server-side replay verification (Dart engine/PRNG/seeder ported to a TS edge function)— A money-tied public leaderboard must be cheat-resistant from day one; determinism makes it feasible.
  • Global per-tier daily leaderboard UI— The headline competitive feature.
Full — target outcome ×8
  • Friend codes + invite (deep) links— Primary, permission-free path to a friends leaderboard; inherently viral.
  • Contacts matching (hashed phone/email, server-side match)— Classic 'find friends who play'; needs only the contacts permission.
  • Outbound share cards to IG/X/anywhere— Realistic growth lever since friend-import is impossible on IG/X; reuses score_share_screen.
  • Friends/local leaderboard UI— Ranks your connected friends by today's score.
  • Streak system surfacing + milestones + rewarded-ad streak freeze— #1 retention lever; streak is already tracked but unused.
  • Local daily-reminder + streak-expiring notifications— Biggest DAU driver and $0 (no backend push needed).
  • Achievements / badges— Cheap, sticky long-term goals; shareable.
  • Extended leaderboards (weekly / monthly / all-time)— Goals beyond the single day; simple Supabase aggregation.
Stretch — if time permits ×3
  • Practice / unlimited off-leaderboard mode
  • Extra rewarded-ad placements (hint / reveal-next-drop)
  • Cosmetic tile themes (unlock via streaks/achievements or rewarded ad)
Out of scope — said no on purpose
  • Facebook friends import (user_friends OAuth) — Requires Meta App Review and only returns mutual app users; deferred to keep scope and review burden down.
  • Instagram / X friend-graph import — No supported friend-graph API on Instagram; X follower endpoints are paid-only. Replaced by outbound sharing.
  • Replay / spectate top runs — Nice-to-have (and near-free given recorded moves) but explicitly cut from this round.
  • Real-time multiplayer — This is async daily competition by design, not live PvP.
  • Paid Supabase tier / managed push (FCM) infrastructure — Stay at $0 until player count justifies an upgrade.
  • Email/password accounts and cross-device recovery — Anonymous-first; account linking is a future enhancement.
Future — someday, maybe
  • Replay & spectate the day's top runs (architecture already records move sequences)
  • Facebook login + friends import once an audience justifies Meta review
  • Account linking for cross-device identity recovery
  • Push notifications when a friend beats your score (needs FCM)
  • Cosmetics store / seasonal cosmetic rewards
  • Seasons with resets and seasonal badges
